# PVF Delivery Map — agent guide

See `CLAUDE.md` for the full rules (single source of truth for agents) and
`docs/current-state.md` for current status. Summary:

- `docs/` = deployed static site (GitHub Pages), no build step, no framework.
- `docs/zones.geojson` / `docs/zips.json` are generated by
  `builder/build_zones.py` — edit the script's ZIP lists, re-run, commit.
- Delivery date logic in `docs/app.js` must stay in sync with GrazeCart
  schedule overrides AND the pvf-order-planner repo's `app.js`.
- Customer-facing copy follows the PVF brand voice.
- Pushing to main publishes to the live site — get Brian's approval first.
